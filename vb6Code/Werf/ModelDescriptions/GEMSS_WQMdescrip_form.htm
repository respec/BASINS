General Description

The GEMSS software uses GLLVHT (Generalized, Longitudinal-Lateral-Vertical Hydrodynamic and Transport) which is a state-of-the-art three-dimensional numerical model that computes time-varying velocities, water surface elevations, water quality, chemical and sediment constituent concentrations in rivers, lakes, reservoirs, estuaries, and coastal water bodies. The computations are done on a horizontal curvilinear and vertical grid that represents the water body bounded by its water surface, shoreline, and bottom. The water surface elevations are computed simultaneously with the velocity components. The water quality constituent concentrations are computed from the velocity components and elevations. Included in the computations are boundary condition formulations for friction, wind shear, turbulence, inflow, outflow, surface heat exchange, water quality, chemical and sediment kinetics. The model can be used to analyze water body system dynamics and predict the impacts of actual events or possible design or management alternatives. The model also uses non-hydrostatic condition in the vertical direction to simulate high discharge flows.

The flow and constituent fields are discretized in time, and the computation marches forward in time steps of 100 s to 900 s, computing the dependent variables throughout the grid at each of these steps. To march the calculations through time, boundary condition data consisting of meteorological data, inflow rates, temperatures, and constituent concentrations; and outflow rates are required.

The GEMSS software is designed in a modular fashion for easy coupling of existing as well as other user defined models. Currently the system has five modules:

    1. HydroDynamic Module - HDM
    2. Water Quality Module - WQM
    3. Sediment Transport Module - STM
    4. Particle Tracking Module - PTM
    5. Oil and Chemical Spill Module – OCSM

The inputs to these modules are obtained from their respective control files and the outputs are designed for easy uploading into existing 2-d and 3-d display modules.

The unique design of GEMSS suite of models gives the user the power of writing adaptation routines to introduce different initial conditions, time variant boundary conditions, replace existing algorithms for source and sink computations related to water quality, sediment transport etc. and nonstandard features or customize the output. In this scheme the main kernel of GLLVHT-HDM behaves like a black box.

The simplest water quality model is from the EPA supported EUTRO5 model that was assembled from earlier models in 1983. This model simulates the interaction of eight water quality state variables, i.e., the dissolved forms of ammonia nitrogen, nitrate nitrogen, inorganic phosphorus, carbonaceous BOD, organic nitrogen, organic phosphorus, phytoplankton and dissolved oxygen. They can be considered as four interacting systems: phytoplankton kinetics, the phosphorus cycle, the nitrogen cycle, and the dissolved oxygen balance. EUTRO5 allows for sources and sinks of NH3 and DO from and to the sediments depending on the settling of CBOD and phytoplankton.

The next most complex model is the WQ3D model, which is an extension of EUTRO5. WQ3D includes both the dissolved and particulate forms of nitrogen, phosphorus and CBOD. It also allows for properly including zooplankton excretion contributions through the particulate forms. Settling is applied only to the particulate components of the constituents. It also allows for sources and sinks of NH3 and DO from and to the sediments, and can be coupled with higher order sediment models. This level of modeling is near state of the art and is being used on many estuaries including the Delaware River Estuary.

The most complex water quality model is the carbon-based model WQ3DCB which is built on WQ3D. This model includes different forms of organic carbon that can be related to sediment exchange processes. WQ3DCB simulates five interacting sub-systems: net phytoplankton production, the phosphorus cycle, the nitrogen cycle, the dissolved oxygen balance, and the particulate organic carbon balance. This model is similar to the model incorporated in the Corps of Engineers ICM model of the Chesapeake Bay. This model was applied and approved for permitting of a WWTP discharge in the Budd Inlet of Puget Sound.

All three models include auxiliary functions for light attenuation as a function of surface solar radiation, phytoplankton concentration (self-shading), and depth; and reaeration rate as a function of wind speed temperature and salinity. Relationships for sediment exchange for ammonia, nitrate nitrogen, inorganic phosphorus, and sediment-oxygen demand (SOD) have been developed from a number of field studies.

The choice of the level of water quality modeling depends on the problem being studied. If a traditional problem of a BOD-NH3 waste water discharge effects on dissolved oxygen were being examined, then the EUTRO5 model might be sufficient. However, for reproducing nutrient cycling and algal growth in most situations, the WQ3D level of model with particulates would be required. Further, if coupling with sediment water quality and exchanges with the sediment were important, then the WQ3DCB model would be important. Recently, WQ3DCB has been revised to include the dinoflagellates as a separate algal group with its own growth and loss rate terms and the ability to undergo vertical migration in response to light. The vertical migration is obtained from three equations which represent temperature and light dependence, and sinking due to buoyancy using Stokes’ law. These equations vectorially add up to the total swimming speed which is applied in the vertical direction. If field data is available for sinking rates, then it can be used in the computation of vertical migration speed instead of Stoke’s law.

There are three approaches currently used in WQM to link the sediment benthic processes with the WQ3DCB model.

  • Benthic rates from field data

  • Empirical exchange rates used in other estuary cases

  • Coupled to DELFT’s OMEXDIA (Ocean Margin Exchange Nutrient Diagenesis Model) model (Soetaert, et al. 1996a,b)

There are numerous reasons for using a model as seemingly complicated as WQ3D or WQ3DCB in almost any application beside the fact that inclusion of the particulate components as well as the dissolved components is state of the art. In most studies it is necessary to separate out the natural processes from the effects of incremental changes due to the wastewater discharge. Modeling nutrient cycling based only on the dissolved components of nitrogen and phosphorous may show some changes but can give inconsistent results with the loading changes of the discharges. Also, the simpler models can be extremely sensitive to the rate parameters. When particulates are included in the water quality modeling, realistic seasonal cycles and responses to changes in loading can be reproduced, and there is less sensitivity to the rate parameters.

The particulate components represent a large portion of the background aquatic chemistry due to zooplankton cycling and when included in the water quality modeling realistic seasonal cycles can be reproduced. The responses to changes in waste loadings and watershed runoff, which are mostly dissolved constituents, can be reproduced more accurately and there is less sensitivity to the rate parameters.

The water quality model can furthermore simulate the distribution and decay of total and fecal coliform bacteria. The unique design of GEMSS modular approach allows for customization of water quality module to be site specific.

Application History

The theoretical basis of the GLLVHT model kernel available in GEMSS was first presented in Edinger and Buchak (1980) and subsequently in Edinger and Buchak (1985). GLLVHT provides three-dimensional, time-varying simulations of rivers, lakes, impoundments, estuaries and coastal water bodies. The GLLVHT model has been peer reviewed and published (Edinger and Buchak, 1995; Edinger, et al., 1994 and 1997). The fundamental computations are an extension of the well known longitudinal-vertical transport model (GLVHT) that was developed by J. E. Edinger Associates, Inc. beginning in 1974 and summarized in Buchak and Edinger (1984). This model forms the hydrodynamic and transport basis of the Corps of Engineers' water quality model CE-QUAL-W2 (U. S. Army Engineer Waterways Experiment Station, 1986).

The GEMSS software has been applied to solve a wide variety of water quality problems and are listed below.

Water quality

  • BOD through nutrient cycles and algae; water body-specific selection of components for modeling from a range of water quality processes, sediment exchange processes.

  • Inorganic chemistry; carbonate balances, sediment interactions.

  • Eutrophication studies; algal growth patterns, nutrient distributions.

  • Sediment transport; rates of scour, resuspension, and settling, effects of structures and facilities on sediment deposition patterns.

  • Toxics; chlorine minimization, heavy metals distributions.

  • Effects of discharge alternatives; waste load allocation, changes in effluent characteristics, changes in outfall locations, DO deficits computations, Interactions with other waste sources.

  • Effects of changes in drainage basin land use, effects on inflow water quality ,non point sources, basin wide TMDL studies.

Model Use Features

The GEMSS software is an integrated system of modules that are designed for easy accomplishment of 3-d modeling of a specific water body by the end-user.

Grid Generation Module

The grid generator module allows the user to create both rectilinear and boundary fitted grids. In the case of rectilinear grids, the user identifies the sub domain of interest (corner points) and selects the appropriate grid size (map units) or fixed number of grids in west-east and north-south directions. An automated gridding algorithm is then used to generate a land-water-shoreline type grid system. At the conclusion of grid generation, the user may edit the computer produced grid to better conform to the shoreline or represent openings to restricted passages (e.g. between islands, narrow inlets, etc.).

In the case of boundary fitted grids, the user sets up a series of boundary control points along the shorelines and open boundaries. These control points are then used to generate boundary fitted grids. The grids then can be edited to make it near orthogonal depending on the complexity of the water body. The grid resolution can be converged around region of interest (decrease grid cell size) or diverged around non-significant areas (increase grid cell size) using a set of attracting and repelling relations that are included in the grid generation module.

Numerous editing options are available so that the user can add features that are not given on the base map or to locate proposed man-made structures (e.g. islands, groins/jetties, passages, inlets, harbors, etc.).

Once completed a bathymetric file is automatically generated and stored under a user selected grid file name. The final grid files contains information about the grid coordinates of each cell, depth and land or water flag for each cell. It also contains information about vertical layer elevation and thickness.

Data Display Module

The data display module allows the user to manage and analyze both field and model output. The analysis can be performed both in 2-D and 3-D. The 3-D visualization module called Qual View allows the user to display bathymetry as a wire frame, color coded, or shaded images. Selecting an output file from the 3-D model, the user can then superimpose 3-D contour surfaces for any observed or computed constituent, e.g., temperature or dissolved oxygen. The combined images can be zoomed, rotated, or redrawn using simple mouse point-and-click procedures. A window can also be opened to show traditional two-dimensional contours on any selected horizontal or vertical transection slices. Qual View uses digitized bathymetric data, which can be generated from contour maps and/or soundings. Qual View reads site geometry from a variety of digitized sources: USGS quads, navigation maps, GIS data, or depth data collected as soundings or along transection slices. It also displays and animates any number of time series current-meter data and model predicted velocity fields.

The 2-d visualization module allows user-controlled visualization and analysis using a set of data managers known as Slice, Contour, Snapshot, Time Series, Vertical Profile Managers. The Slice Manager allows the user to create vertical slicing in any direction and create animation files. The Contour and Snapshot Managers are used to create surface, bottom and at any vertical location contouring. Also available in the Contour and Slice Managers is volume and area calculation tools within specified contour levels. The Time-Series Manager allows the user to display time series data from different locations in the water body for different constituents. The Vertical Profile Manager allows the user to display vertical profiles of different constituents at different locations of the water body. All these managers have a wide variety of options/properties to customize plot sheet figures either for presentation purposes or report writing. VCR type of controls are used to create animation files in all these managers. The animation files allow users to show important hydrodynamic and transport events over time. Animations of this type are invaluable in analyzing management scenario results. Multiple model outputs and field data can be compared in all of these managers using either the same plot sheet or different plot sheets. Overlay of GIS themes on these plot sheets are allowed for environmental impact analysis. Template and project settings give the user ability to save preferred settings for use in other projects. The visualization module has undergone third-party review with respect to the coding and plume dimension computations.

Control File Generator Module

Input data to run a suite of models available in GEMSS is achieved though the use of GIS based control file generator. A simple point and click procedure will allow the user to define various boundary conditions on the water body. A wide variety of boundary conditions such as inflows, outflows, open full boundary, open Dirichlet boundary, sediment release, areal precipitation, powerplant, interior boundaries, specific momentum, vertical releases and bubblers can be selected using the spreadsheet dialog control. The input data can be either constant, time varying or simple harmonic. The time varying and simple harmonic types of data are accessed through a set of input files that are written in a standard .csv (comma separated value) format. Also, the control file generator allows the user to control the output frequencies, model simulation times, model hydrodynamic parameters etc.

Geographical Information System (GIS) Module

The GIS module allows the user to input, store and load data as layers in .SHP format used in ArcView. These layers can be switched on or off, change color or display attribute information on each model output window for analysis using GIS layer floating tool. The overlaying feature is often useful in analyzing model predictions. Also added in this module is annotation. The annotation consists of point, polyline or polygon objects with a wide variety of properties such as color, text, etc. The user can also link any text, data or graphics files to the annotation objects for more effective presentation. The model and annotation data can be exported to a standard ASCII file and ArcView .SHP file.

Field Data Module

The field data module allows the user to load and display field measurements. The field data can be loaded in contour, time series and vertical profile formats. The field data can be overlaid on model results for model calibration and verification. It can also be overlaid on GIS theme layers. The field data information is also used in setting up the different grid resolution in the model grid.

Users Manual

The user manual is embedded into the software using HTML help control technology. It is a complete documentation of all the components available in the software including model theory. The manual has a series of ‘how to do…….’ sections which provides a step by step approach for the easy usage of the software.
